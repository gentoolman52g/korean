제품 요구사항 정의서 (PRD - Product Requirements Document)
1. 개요 및 목표
1.1. 배경:
사내 AI 자동화 앱 '미소'를 통해 임직원들이 제출하는 다양한 자료(업무 매뉴얼, 보고서 등)를 RAG 시스템의 지식 베이스로 활용하고자 합니다.
RAG 시스템의 검색 정확도와 답변 품질은 입력되는 데이터의 정제 수준에 크게 좌우됩니다.[2][3] 부정확하거나 노이즈가 많은 데이터는 AI의 환각(Hallucination) 현상을 유발할 수 있습니다.[3][4]
1.2. 문제 정의:
API를 통해 수신된 원본 텍스트는 중복 내용, 불필요한 서식, 깨진 문자 등을 포함하고 있어 RAG 시스템에 바로 등록하기에 부적합합니다.
수동으로 데이터를 정제하는 것은 비효율적이며 일관성을 유지하기 어렵습니다.
1.3. 목표:
RAG 등록에 최적화된 형태로 텍스트를 자동 전처리하고, 사용자가 그 결과를 검토 및 수정하여 최종 데이터를 생성할 수 있는 워크플로우를 제공합니다.
이를 통해 RAG 시스템의 기반 데이터 품질을 향상시키고, AI 답변의 신뢰도를 높입니다.
2. 사용자 및 사용 시나리오
2.1. 주요 사용자: '미소' 앱을 통해 자료를 제출하고 RAG 시스템에 등록하는 모든 임직원
2.2. 사용자 시나리오 (User Flow):
임직원이 '미소' 앱에 문서를 업로드하면, 문서 내용이 파싱되어 텍스트로 변환된 후 본 시스템의 API로 전송됩니다.
시스템은 API로 받은 텍스트를 자동으로 전처리 및 청킹합니다.
임직원은 전처리 결과가 표시된 웹 페이지 링크를 받습니다.
임직원은 해당 페이지에서 전처리된 내용을 확인하고, 필요시 문맥에 맞게 수정합니다.
수정이 완료되면 '다운로드' 버튼을 클릭하여 최종 .txt 파일을 내려받습니다.
임직원은 다운로드한 파일을 사내 RAG 시스템에 직접 등록합니다.
3. 기능 요구사항
ID	기능명	상세 설명	우선순위
FR-01	API 데이터 수신	- HTTP POST 요청을 수신할 수 있는 API 엔드포인트 제공<br>- 입력 데이터 형식: JSON (key: "text")	필수
FR-02	자동 전처리 모듈	- 중복 문단 제거: 완전 일치하는 문단을 찾아 중복을 제거<br>- 공백/줄바꿈 정규화: 연속 공백, 불필요한 줄바꿈 제거<br>- 불용 문자 제거: 검색에 불필요한 특수 문자, 제어 문자 등 제거	필수
FR-03	텍스트 청킹 모듈	- 전처리된 텍스트를 최대 4000자 단위의 청크로 분할<br>- 문장이나 문단이 중간에 잘리지 않도록 분할 지점 조정<br>- 각 청크를 '@@@' 구분자로 연결	필수
FR-04	미리보기 및 편집 UI	- 전처리 및 청킹 결과를 보여주는 웹 페이지<br>- 전체 텍스트를 수정할 수 있는 대형 텍스트 편집기(WYSIWYG 불필요)<br>- 직관적인 UI/UX 제공	필수
FR-05	TXT 파일 다운로드	- 편집기 내의 최종 텍스트를 .txt 파일로 다운로드하는 기능<br>- 파일명은 "preprocessed_data.txt" 와 같이 기본값 설정	필수
